电赛的遗憾，未来的期许

2024/8/8 心得体会 C 竞赛

暑假留了一个月的校准备电赛，最后什么也没捞到。虽然早有预期但终究是我心里的疙瘩，故述往事、思来者。
希望能给我的学弟学妹留下点什么。

我和竞赛

我必须非常感谢一个人 Aye10032 和 xushengyuan,在我初中的时候给我种下了有关竞赛的一切向往，直到现在也还是我大学生活的白月光
在竞赛上，我可以算是一个十足的新人，总是缺少动力让我去争取机会。我刚入学的时候就和班主任说因为英语对冲刺考研意愿不高，想去追求我的白月光：竞赛。结果第一学期英语就挂科了，这其中有疫情、老师针对（平时分没考的高）种种因素，但结果就是我萎了一年，一直到大二下才真正开始准备竞赛。但这时考研的压力也攀上我的脖颈。
我一直认为我可以做一个垂帘听政的幕后角色，但事实上我已经等不到别人发现我，再到能把幕后交给我了。
所以啊，人生中的一些东西还是要自己去交涉的，切不要因为社恐误了自己。
如果你想参加竞赛，大一上，最晚大一下就可以找班主任了解一下，如果班主任对这方面不感兴趣，你也可以在自己院里找找一般都有实验室，敲门进去问问。

电赛

作为第一年打电赛的小白，我也没多少经验可以分享，几点心得写出来与大家共勉。

  • 每个组都要有个组长（主催），至少一个实验室里做同一个项目的要有一个人了解整个项目的框架。
  • 出赛题之前的准备是很重要的，能学的东西有很多。以我打的无人机赛题举例

    • 飞控和飞手一般为同一人，实际上的任务是最重的：

      1. 对无人机的接线、传感器、设置、复位都要有很清晰的认知
      2. 在熟悉无人机的基础上，要清楚无人机的各物理部件的作用，能拆散重装一遍为佳
      3. 清楚飞控的技术细节，对外设、视觉的框架清晰
      4. 最好能够用自己的代码复刻一遍往年赛题，即使来不及，也要知道我每个部分怎么写，写在哪。例如通信一般是写在中断里的，对某个模块进行调试可以单写一个 SDK

    • 视觉端可能是赛前看上去最闲的部门，OpenMV一般都有现成的代码。视觉能做的只有对题调试，但实际上并不是：

      1. 视觉重在平时的积累，非常要求熟练度
      2. 视觉与飞控端的通信，视觉端的发包，飞控端的解包都需要提前准备好。

    • 外设的任务最重也最没有方向，但无论如何，我认为以下几点是一定要做好的

      1. 无线通信模块，最好是8266和蓝牙模块各备一手。
      2. 需要会一个大屏幕，我个人推荐串口屏+Arduino，当然那个顺手用哪个
      3. 以上所有部件之间的通信，和飞控的通信这都是要提前准备好的。

当然每个组的分工不一样，除了我这样的分法，也可以按硬件、软件、软硬件沟通这样分，但一定要有一个主催，一个优秀的主催对项目的推进帮助极大。怕的不是某个模块搞不懂（驱动不了）怕的是有人眼高手低，不愿尝试。
从比较功利的角度来说，做一个假的先拿到保底分没有问题。只要能过自己和他人心里那关，但我不会这样，做不出来就是做不出来，自己能力不够。

未来的展望

对于现在来说，大二到大三打比赛实在是晚了一步，但我并不想放弃。反正我英语差，考研也不要求多好的学校了，还是希望尝试一下。
深感自己的技术简陋，正在全力学习中，当然黑神话还是要玩的。
目前对以下几个比赛有点兴趣：

  1. 中国大学生计算机设计大赛
  2. 全国大学生电子设计竞赛（国赛再冲一次）

结语
